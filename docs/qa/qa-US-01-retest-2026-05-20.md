## Bug novo encontrado durante teste funcional

apps/api/src/modules/finances/infra/prisma-despesa.repository.ts:26-47 ❌ CRÍTICO: `origemId` não é persistido no `create()`. O campo `origemId` está AUSENTE no objeto `data` passado ao `prisma.despesa.create()`. Parcelamentos e séries recorrentes dependem deste campo para vincular despesas entre si.

Reprodução:
```
POST /api/despesas
{ "abaId": 1, "mesRef": "2026-05", "descricao": "Parcela 1/3", "valor": 100, "origemId": 10 }
→ 201: { ..., "origemId": null }   ← origemId ignorado silenciosamente
```

Impacto: `DELETE ?serie=true` não consegue encontrar despesas da mesma série via `origemId`. Criação de parcelamentos (totalParcelas > 1) retorna parcelas desvinculadas.

## Bug encontrado (MÉDIO)

apps/api/src/modules/finances/application/use-cases/delete-orcamento.use-case.ts:6 ⚠️ MÉDIO: `DELETE /api/orcamentos/:id` retorna 500 quando ID não existe em vez de 404. O use case chama `this.orcamentoRepo.delete(id)` diretamente sem verificar existência prévia — Prisma lança `PrismaClientKnownRequestError P2025` que resulta em 500.

Reprodução:
```
DELETE /api/orcamentos/99999 → 500 { "error": "Internal server error" }
(esperado: 404)
```

Fix: adicionar `findById(id)` + `if (!existing) throw HttpError.notFound(...)` antes do delete.
